# Edu CoreX Update Release Rules

## Android OTA updates

Use Capgo OTA for React-only changes:

- React pages and routes
- CSS and animations
- JavaScript logic
- Static web assets copied into `dist`

Build and publish:

```powershell
npm run android:ota:bundle
$env:CAPGO_TOKEN="your-capgo-token"
npm run android:ota:publish
```

The installed APK downloads OTA bundles when online and keeps using the last good bundle when offline.

## Windows desktop updates

Electron desktop updates are delivered through GitHub Releases.

Before publishing a desktop update:

```powershell
npm run version:patch
npm run publish:win
```

The GitHub Release must contain the NSIS installer, `latest.yml`, and blockmap file generated by electron-builder.

## Native Android updates

Build a new APK when changing:

- Native Android code
- Capacitor plugins
- Android permissions
- `AndroidManifest.xml`
- Package id
- Gradle configuration
- App icon or splash resources

For native APK releases, bump Android `versionCode` and `versionName`, then rebuild the APK.
